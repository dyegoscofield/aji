# Checklist de Compliance para Novas Features

## Antes de Lançar Qualquer Nova Funcionalidade

```
□ A feature poderia ser interpretada como "consultoria jurídica"?
□ O usuário poderia entender que está recebendo um parecer?
□ Há disclaimer adequado na UI?
□ O advogado responsável técnico (Julio) revisou?
□ O sistema força escalada em casos de alta complexidade?
□ Não há promessas de resultado?
□ A feature é apenas para orientação, não para representação?
□ Textos de marketing usam apenas termos seguros?
□ Dados sensíveis estão protegidos conforme LGPD?
□ Endpoint de portabilidade cobre os novos dados?
```

## Quando Acionar Este Checklist

1. **Sempre** antes de lançar nova feature de orientação jurídica
2. Ao revisar textos de landing page, emails de marketing
3. Ao definir o que o AJI pode ou não responder
4. Ao revisar termos de uso e política de privacidade
5. Se algum usuário reclamar que foi orientado incorretamente
6. Se a OAB entrar em contato ou houver notificação
7. Para revisar o system prompt do modelo de IA
